如果说“缓存雪崩”是由于大面积 Key 失效导致的**全面坍塌**，那么“缓存击穿”（Cache Breakdown）就是针对**单个超级热点 Key** 的**定点爆破**。

---

### 1\. 核心结论 (Core Conclusion)

**缓存击穿是指：某一个被高并发访问的热点 Key，在过期的那一瞬间，大量请求同时发现缓存失效，于是这些请求穿过缓存，像“钻头”一样同时轰击数据库。**

-   **场景特征**：单个 Key、高并发、恰好过期。
    
-   **后果**：数据库连接池瞬间被占满，CPU 飙升，导致数据库卡死或宕机。
    

---

### 2\. 深度方案拆解 (Detailed Breakdown)

解决缓存击穿，业界主要有两种完全不同的哲学：

#### 方案一：互斥锁 (Mutex Lock) —— 追求“强一致性”

-   **原理**：当缓存失效时，不立即去查数据库，而是先用 `SETNX`（或 Redisson 的锁）去获取一个互斥锁。拿到锁的那个线程去查库并回写缓存，没拿到锁的线程自旋重试或等待。
    
-   **优点**：保证了数据库只会被请求一次，数据一致性高。
    
-   **缺点**：性能受限。在高并发下，大量线程在等待锁，吞吐量会有所下降。
    

#### 方案二：逻辑过期 (Logical Expiration) —— 追求“高可用性”

-   **原理**：
    

1.  Redis 物理上不设置过期时间（永久有效）。
    
2.  在 Value 内部封装一个“逻辑过期时间”字段。
    
3.  查询时，如果发现逻辑时间已过期，**先返回旧数据**，同时开启一个**异步线程**去查询数据库并更新缓存。
    

-   **优点**：性能极高。请求永远不会被阻塞，始终能秒回数据。
    
-   **缺点**：存在短暂的数据不一致。在异步线程更新完之前，用户拿到的都是旧值。
    

---

### 3\. 面试官视角 (Interviewer's Lens)

#### 问：这两种方案如何选型？

-   **答案**：
    
-   如果业务涉及**钱、库存、价格**等敏感信息，必须用**互斥锁**，宁可慢点也要准。
    
-   如果业务是**新闻资讯、热搜榜单、社交状态**，优先用**逻辑过期**，保证用户体验顺畅，稍微看到点旧新闻没关系。
    

#### 问：互斥锁方案中，如果拿到锁的线程挂了，不就死锁了吗？

-   **答案**：加锁时必须设置**过期时间**（Lease Time）。即使线程崩溃，锁也会在几秒后自动释放，其他线程可以继续尝试。
    

#### 问：为什么不直接把热点 Key 设置为“永久有效”？

-   **答案**：虽然可以，但并不灵活。
    

1.  内存会持续膨胀。
    
2.  如果数据库数据变了，缓存永远不会更新，除非手动清理。逻辑过期方案本质上是“手动维护的永久有效”。

### 4\. 代码演示：互斥锁实现逻辑 (Java)

```java
public String getData(String key) {
    // 1. 从缓存读
    String value = redis.get(key);
    if (value != null) return value;

    // 2. 缓存失效，尝试获取分布式锁
    String lockKey = "lock:" + key;
    if (redis.setnx(lockKey, "1", 10, TimeUnit.SECONDS)) {
        try {
            // 双重检查 (Double Check)，防止其他线程刚才已经写回了缓存
            value = redis.get(key);
            if (value != null) return value;
            
            // 3. 查数据库
            value = db.query(key);
            // 4. 回写缓存
            redis.set(key, value, 30, TimeUnit.MINUTES);
        } finally {
            // 5. 释放锁
            redis.delete(lockKey);
        }
    } else {
        // 6. 没拿到锁的，休眠一会再重试
        Thread.sleep(100);
        return getData(key); 
    }
    return value;
}
```

### 5\. 总结 (Summary)

1.  **击穿 vs 雪崩**：击穿是“单点爆破”，雪崩是“全面瘫塌”。
    
2.  **核心解法**：
    

-   **互斥锁**：保一致性，损性能。
    
-   **逻辑过期**：保性能，损一致性。
    

3.  **避坑指南**：互斥锁一定要带过期时间，查库前记得做 Double Check。
