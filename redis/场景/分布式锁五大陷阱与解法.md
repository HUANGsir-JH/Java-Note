
在分布式环境下，分布式锁看似简单（加锁、执行、解锁），但实际上潜伏着许多“致命”陷阱。以下是面试中最常被问到的 5 个核心问题及其解决方案。

---

### 1. 核心结论 (Core Conclusion)

分布式锁的三大核心挑战是：**“锁提前过期”**、**“解了别人的锁”**、以及 **“Redis 挂了锁丢了”**。
解决这些问题的工业级标准方案通常是：**Redisson 框架（看门狗机制）** + **Lua 脚本** + **（极端场景下的）Redlock 或 Zookeeper**。

---

### 2. 问题与解决方案 (Detailed Breakdown)

#### 问题 ①：业务没跑完，锁自动过期了 (The Expiration Problem)
* **现象**：设置锁 30 秒过期，但业务因为 GC 或复杂逻辑跑了 40 秒。第 30 秒时锁自动释放，其他线程趁虚而入，导致并发安全问题。
* **解决：看门狗机制 (Watchdog)**。
 * **原理**：在加锁成功后，启动一个后台定时任务（守护线程），每隔一段时间（如 10 秒）检查业务是否完成。若未完成，自动延长过期时间。
 * **工具**：Java 中 **Redisson** 已经内置了此功能，无需手动实现。

#### 问题 ②：误删了他人的锁 (The "Steal" Release)
* **现象**：线程 A 执行慢导致锁过期。此时线程 B 拿到锁。接着 A 执行完了，顺手调了 `DEL`，把 B 正在使用的锁给删了。
* **解决：唯一标识 + Lua 脚本**。
 * **原理**：加锁时存入一个唯一的 `UUID`（或线程 ID）。解锁时，先 `GET` 出来对比是否是自己的 UUID，匹配才 `DEL`。
 * **关键**：为了防止“比对成功”和“删除”之间发生切换，这两步必须写成 **Lua 脚本**交给 Redis 原子执行。

#### 问题 ③：Redis 节点宕机导致锁丢失 (Master-Slave Lag)
* **现象**：在 Redis 主从集群中，Master 节点刚写完锁就挂了，还没来得及同步给 Slave。Slave 变成新 Master 后，锁数据丢失，导致两个客户端同时拿到锁。
* **解决 1：Redlock（红锁）**。
 * **原理**：向多个（通常是 5 个）独立的 Redis 节点同时申请锁。只有超过半数（3 个）成功才算加锁成功。
* **解决 2：切换为强一致性组件 (Zookeeper)**。
 * **建议**：如果业务要求绝对的安全性（如金融扣款），建议弃用 Redis 锁，改用基于 CP 模型的 **Zookeeper** 实现。

#### 问题 ④：不可重入性 (Non-reentrancy)
* **现象**：同一个线程在获得锁后，再次调用该逻辑（递归或嵌套调用），由于锁已被自己占用，导致死锁。
* **解决：Hash 结构 + 计数器**。
 * **原理**：Redis 数据结构改为 Hash。Key 是锁名，Field 是线程 ID，Value 是加锁次数。
 * **逻辑**：每次加锁次数+1，解锁-1，为 0 时彻底删除 Key。

#### 问题 ⑤：获取锁失败后的“性能浪费” (High Contention)
* **现象**：大量线程在 `while(true)` 循环中死命重试获取锁，把 CPU 跑满。
* **解决：发布/订阅模式 (Pub/Sub)**。
 * **原理**：当锁释放时，Redis 发布一条消息。正在等待的线程通过订阅（Subscribe）感知到锁已释放，再去争抢，避免无意义的自旋消耗。

---

### 3. 面试官视角 (Interviewer's Lens)

#### 问：为什么分布式锁不推荐用 SQL 数据库（MySQL）来实现？
* **答案**：
 1. **性能差**：频繁的磁盘 IO 撑不住高并发。
 2. **死锁风险**：没有自动过期机制。如果获得锁的服务挂了，锁就永远留在了表里。
 3. **功能简陋**：不支持看门狗、阻塞唤醒等高级特性。

#### 问：Zookeeper 锁和 Redis 锁到底选哪个？ (The "Cap" Choice)
* **答案**：
 * **追求性能**：选 Redis（AP 模型）。它允许极小概率的并发冲突，但速度飞快。
 * **追求绝对安全**：选 Zookeeper（CP 模型）。它利用临时顺序节点和 Watch 机制，能保证强一致性，但性能略低于 Redis。

### 4. 总结 (Summary)

| 问题 | 解决方案 | 核心关键词 |
| :--- | :--- | :--- |
| **锁提前过期** | **Redisson Watchdog** | 锁续期 |
| **误删锁** | **UUID + Lua 脚本** | 原子删除 |
| **主从同步丢失** | **Redlock / Zookeeper** | 强一致性 |
| **不可重入** | **Hash + Counter** | 计数器 |
