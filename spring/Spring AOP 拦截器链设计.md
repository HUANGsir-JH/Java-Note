如果一个方法匹配了多个切点（比如你既给它加了日志切面，又加了权限切面，还加了 Spring 自带的事务切面），Spring 并不会为你生成一叠厚厚的“多重代理”，而是采用了一种更优雅的设计：**拦截器链（Interceptor Chain）**。

你可以把它想象成**一层又一层的洋葱**，或者**一个接一个的关卡**。

### 1\. 核心机制：拦截器链（Chain of Responsibility）

当你调用代理对象的方法时，代理对象内部并不是直接去找原始对象，而是先拿出一个**列表**。这个列表里存放了所有匹配这个方法的“通知（Advice）”。

Spring 会把这些通知统一封装成 `MethodInterceptor`（方法拦截器）对象。

### 2\. 执行流程：神奇的 `proceed()`

这是理解 AOP 多个切面叠加的关键。执行过程就像一个**递归**（或者说回文结构）：

1.  **进入代理：** 代理对象开始执行拦截器链。
    
2.  **第一个拦截器（比如：日志）：**
    

-   它先执行自己的“前置逻辑”（打印日志开始）。
    
-   然后它会调用一个神奇的方法：`invocation.proceed()`。这个方法的意思是：“我不决定接下来的事，请交给链条里的下一个拦截器吧”。
    

3.  **第二个拦截器（比如：权限）：**
    

-   它接过球，执行“前置逻辑”（检查权限）。
    
-   它也调用 `invocation.proceed()`。
    

4.  **第三个拦截器（比如：事务）：**
    

-   开启事务。
    
-   调用 `invocation.proceed()`。
    

5.  **到达终点：** 此时链条里没拦截器了，Spring 终于调用了**原始对象**的真实方法。
    
6.  **原路返回（回溯）：**
    

-   事务拦截器：提交/回滚事务。
    
-   权限拦截器：清理上下文。
    
-   日志拦截器：打印日志结束。
    

**总结：** 所有的切面逻辑都在**同一个代理对象**内部，通过一个列表维护，并按照顺序依次嵌套执行。

### 3\. 谁先执行？（顺序问题）

既然是一个链条，就肯定有先后。Spring 默认有一套规则，但你也拥有控制权：

-   **默认顺序：** 如果没有指定，通常按照切面类的名字字母顺序，或者 Spring 内部加载的顺序。
    
-   **手动指定：**
    
-   使用 `@Order(1)` 注解。数字越小，优先级越高，意味着它在“洋葱”的最外层（最先进入，最后退出）。
    
-   实现 `Ordered` 接口。
    
-   **事务的特殊性：** 通常我们会把事务切面的优先级设得比较低（靠近洋葱心），这样日志和权限切面就能在事务开启之前执行，避免权限不足还去开启数据库连接。
    

### 4\. 代码层面的样子

如果你去看 Spring 源码，核心类叫 `ReflectiveMethodInvocation`。它的 `proceed()` 方法逻辑大致是这样的（简化版）：

```java
public Object proceed() {
    // 1. 如果执行到了链条末尾，直接调用目标方法
    if (this.currentInterceptorIndex == this.interceptors.size() - 1) {
        return invokeJoinpoint(); 
    }

    // 2. 获取下一个拦截器
    Object interceptor = this.interceptors.get(++this.currentInterceptorIndex);
    
    // 3. 调用拦截器。注意：拦截器内部会再次调用此 proceed() 方法，形成递归
    return ((MethodInterceptor) interceptor).invoke(this);
}
```

### 5\. 这种设计的精妙之处

-   **解耦：** 每个切面只关心自己的逻辑，它不需要知道链条里还有谁，只需要调一下 `proceed()` 往下传就行。
    
-   **统一：** 无论是 1 个切面还是 100 个切面，对于调用者来说，拿到的永远是同一个代理对象，接口完全一致。
    
-   **灵活：** 可以在运行时动态地调整拦截器链的顺序。
    

### 总结一下：

如果有多个切面，Spring 会把它们排成一队。代理对象被调用时，会像接力赛一样，一个接一个地执行这些切面逻辑。这种模式在设计模式中叫**责任链模式**，它是 Spring 处理各种“嵌套逻辑”（如 AOP、Filter、Interceptor）的通用套路。
