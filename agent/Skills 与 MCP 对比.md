# Skills 与 MCP 的异同对比

> 您问的应该是 AI Agent 领域的 **Claude Skills** 与 **MCP（Model Context Protocol）**，以下是详细对比：

---

## 🔑 一句话总结

> 如果 AI Agent 是操作系统，**MCP 就是 USB 协议**，**Skills 就是应用程序**

---

## 📋 核心对比表

| 维度  | MCP（Model Context Protocol） | Claude Skills |
| --- | --- | --- |
| **定位** | 标准化连接协议 | 任务能力封装包 |
| **核心能力** | 连接外部系统/数据源 | 编码专业知识/工作流程 |
| **类比** | USB-C 接口标准 | 预装应用程序 |
| **上下文消耗** | 预加载工具定义，成本高（单Server常达10K+ tokens） | 渐进式披露，按需加载，成本低 |
| **工具来源** | 外部 MCP Server | 内置工具 + 自带脚本 |
| **网络访问** | ✅ 支持远程 API/云服务 | ❌ 主要面向本地执行 |
| **分发方式** | URL接入，面向外部用户 | 文件复制，面向内部团队 |
| **适用场景** | 远程数据库、SaaS API、对外服务 | 本地流程、专业方法论、内部工具 |

---

## 🔍 相同点

1.  **目标一致**：都是为了增强 AI Agent 的能力，让其能完成更复杂的任务
    
2.  **均由 Anthropic 推动**：MCP 和 Skills 都出自 Anthropic 生态
    
3.  **支持模块化扩展**：都允许开发者按需添加能力，而非硬编码
    
4.  **可协同使用**：最佳实践是 Skills 编码领域知识 + MCP 连接外部服务
    

---

## ⚡ 关键差异详解

### 1️⃣ 上下文消耗机制（最核心差异）

**MCP 的问题**：

-   连接时必须预加载所有工具定义（名称、描述、参数等）
    
-   一个 MCP Server 通常消耗 10K~18K tokens[[20]]
    
-   连接多个 Server 可能吃掉 30%+ 的上下文窗口，影响 AI 判断准确性
    

**Skills 的优势**：

-   采用**渐进式披露**：启动时只加载 ~100 tokens 的元数据
    
-   完整指令仅在任务相关时才加载
    
-   脚本执行时，代码本身不进上下文，只返回结果[[20]]
    

### 2️⃣ 能力封装方式

```text
MCP 模式：
AI → MCP协议 → 外部工具/API → 返回结果
（AI需要理解工具定义 + 逐步决策执行）

Skills 模式：
AI → 读取SKILL.md → 调用本地脚本 → 返回结果  
（AI只需知道"做什么"，"怎么做"由脚本封装）
```

### 3️⃣ 分发与协作

|     | MCP | Skills |
| --- | --- | --- |
| **安装** | 输入URL或配置Server地址 | 复制SKILL.md到指定目录 |
| **权限** | 可配置细粒度访问控制 | 依赖本地文件系统权限 |
| **共享** | 适合对外提供标准化服务 | 适合团队内部知识沉淀 |

---

## 🎯 如何选择？

### ✅ 优先用 Skills 的场景：

-   读写本地文件、处理PDF/Excel
    
-   执行Git操作、运行代码分析
    
-   封装团队内部工作流程或专业方法论
    
-   只有你自己或小团队使用
    

### ✅ 必须用 MCP 的场景：

-   连接远程数据库、CRM、Jira等SaaS系统
    
-   调用需要认证的第三方API
    
-   需要对外提供标准化AI服务
    
-   跨网络/跨组织的系统集成
    

### ✅ 最佳实践：两者配合

> 用 Skills 编码你的领域知识（"怎么分析数据"），用 MCP 连接外部服务（"从哪里获取数据"）

---

## 💡 形象类比

```text
🏢 公司新员工培训：

MCP 做法：入职第一天把公司所有制度手册、系统文档、
        操作指南全部堆在桌上（预加载，信息过载）

Skills 做法：先给岗位说明书，遇到具体问题时，
          再告诉他"去查XX手册第X章"（按需加载，高效聚焦）
```

## 🔮 趋势展望

随着 Skills 生态成熟，MCP 的角色可能会收窄到\*\*"远程连接"核心场景\*\*，而本地任务、浏览器自动化、数据处理等将由 Skills + 内置工具高效完成。

> 📌 **记忆口诀**：MCP 让 AI 能"碰到"数据，Skills 教 AI 怎么"处理"数据
