结论是：**可以，但必须使用正确的方法。**

如果方式不对，程序会抛出著名的 `ConcurrentModificationException`（并发修改异常）或者导致逻辑错误（漏删元素）。

以下是几种遍历修改场景的详细分析：

---

### 1\. 错误方式：增强 For 循环（foreach）

**结果：抛出异常**

```java
List<String> list = new ArrayList<>(Arrays.asList("a", "b", "c"));
for (String s : list) {
    if ("b".equals(s)) {
        list.remove(s); // 抛出 ConcurrentModificationException
    }
}
```

-   **原因**：foreach 语法糖底层使用的是 `Iterator`。`Iterator` 在遍历时会维护一个 `expectedModCount`（预期修改次数），而 `list.remove()` 只是更新了列表的 `modCount`。下次循环执行 `it.next()` 时，发现两个计数器不相等，于是触发 **fail-fast（快速失败）** 机制，抛出异常。
    

---

### 2\. 危险方式：普通 For 循环（下标遍历）

**结果：不会报错，但会漏掉元素**

```java
for (int i = 0; i < list.size(); i++) {
    if ("a".equals(list.get(i))) {
        list.remove(i); 
    }
}
```

-   **原因**：当你删除了下标为 `i` 的元素后，后面的元素会全部向前移动一位。此时循环进入 `i+1`，原本在 `i+1` 位置的元素现在到了 `i`，它就被**跳过**了。
    
-   **修正方案**：从后往前遍历 `for (int i = list.size() - 1; i >= 0; i--)`。
    

---

### 3\. 正确方式 A：使用 Iterator 的 remove 方法

**结果：安全**

```java
Iterator<String> it = list.iterator();
while (it.hasNext()) {
    String s = it.next();
    if ("b".equals(s)) {
        it.remove(); // 注意：调用的是 iterator 的 remove，不是 list 的 remove
    }
}
```

-   **原理**：`it.remove()` 在删除元素后，会同步更新迭代器内部的 `expectedModCount`，使其与列表的 `modCount` 保持一致，因此不会报错。
    

---

### 4\. 正确方式 B：Java 8 的 `removeIf`（最推荐）

**结果：简洁、安全、性能好**

```java
list.removeIf(s -> "b".equals(s));
```

-   **原理**：这是 Java 8 引入的声明式方法，底层已经处理好了遍历和删除的逻辑，代码最简洁，不容易出错。
    

---

### 5\. 正确方式 C：使用并发容器 `CopyOnWriteArrayList`

**结果：安全，支持在 foreach 中修改**

```java
List<String> list = new CopyOnWriteArrayList<>(Arrays.asList("a", "b", "c"));
for (String s : list) {
    if ("b".equals(s)) {
        list.remove(s); // 不会报错
    }
}
```

-   **原理**：`CopyOnWriteArrayList` 在遍历时是基于原数组的“快照”进行的。你修改（删除/添加）的是新副本，不影响正在进行的遍历。
    
-   **缺点**：每次修改都要复制数组，开销大。
    

---

### 总结与面试建议

| 遍历方式 | 是否推荐修改 | 说明  |
| --- | --- | --- |
| **增强 For (foreach)** | ❌ **绝对不行** | 必报 `ConcurrentModificationException` 异常。 |
| **普通 For (正向)** | ⚠️ **不建议** | 容易漏删元素，除非手动处理下标 `i--`。 |
| **普通 For (反向)** | ✅ **可以** | 从后往前删，下标移动不影响未遍历的元素。 |
| **Iterator** | ✅ **推荐** | 传统的安全做法。 |
| **removeIf (Java 8)** | 🏆 **最推荐** | 语义清晰，代码量最少。 |
| **CopyOnWriteArrayList** | ✅ **可以** | 适用于多线程写少读多的场景。 |

**面试官心声：** 如果你提到 `removeIf`，说明你熟悉现代 Java 开发；如果你能解释 `modCount` 的 fail-fast 机制，说明你深入看过源码。
