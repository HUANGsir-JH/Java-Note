---

### 💡 核心结论
**DDoS（Distributed Denial of Service，分布式拒绝服务攻击）** 本质上是一种**“暴力拆迁”**。攻击者通过控制成千上万台“肉鸡”（僵尸主机），在短时间内向目标服务器发起海量请求，耗尽服务器的 **带宽、连接数、CPU 或内存**，导致合法用户无法访问。

---

### 🔍 深度拆解：常见的攻击手段

DDoS 不是一种单一攻击，而是针对 OSI 七层模型的全方位打击：

#### 1. 网络层/传输层攻击（消耗带宽与内核资源）
* **SYN Flood (最典型)**：利用 **TCP 三次握手** 漏洞。发送大量 `SYN` 包但不回第三次 `ACK`。服务器会维持大量的 **半开连接（Half-Open）**，迅速填满半连接队列。
* **UDP Flood**：向目标端口发送海量大字节 UDP 包。虽然不建立连接，但会塞满服务器的入口带宽。
* **ICMP Flood**：疯狂 Ping 目标，让服务器忙于回复。

#### 2. 应用层攻击（消耗业务逻辑与 CPU）
* **HTTP Flood (CC 攻击)**：模拟真实用户频繁请求耗时操作（如查询数据库、搜索）。连接是合法的，但请求量超过了 Web Server 或 DB 的处理极限。
* **DNS Query Flood**：通过海量域名解析请求压垮授权 DNS 服务器。

---

### 🛡️ 如何防范：多维度防御体系

DDoS 无法完全通过单机代码防御，通常需要**多层联动**：

#### 1. 网络层防御（拒敌于国门之外）
* **SYN Cookie (针对 SYN Flood)**：
 * **原理**：在握手时不直接分配资源，而是计算一个 `Cookie`（序列号）。只有收到合法的第三次 `ACK` 后，才分配内存。
 * **效果**：让服务器在遭受攻击时依然能处理正常连接。
* **Anycast BGP (多点接入)**：将流量分散到全球多个清洗节点，让单点不再承受全部压力。
* **带宽冗余**：这就是所谓的“硬扛”，用 1T 的带宽防御 500G 的攻击。

#### 2. 流量清洗 (DDoS Scrubbing)
* **云厂商高防 (阿里云/Cloudflare)**：流量先经过云厂商的清洗中心。云厂商会通过**特征识别**（比如请求头不规范）或**挑战响应**（比如强制要求执行一段 JS 或回显 Cookie）剔除机器人。

#### 3. 应用层防御
* **限流 (Rate Limiting)**：对单个 IP 的并发连接数和单位时间请求数做硬限制（Nginx `limit_req` 模块）。
* **缓存机制**：对于频繁被刷的接口，尽量走 Redis 缓存，减轻数据库压力。
* **WAF (Web Application Firewall)**：识别异常的 HTTP 行为模式。

### 🧐 面试官视角：进阶陷阱 (The "Interviewer's Lens")

#### 1. “为什么不能在应用层（Java/Go）代码里完全解决 SYN Flood？”
* **核心回答**：因为当请求到达你的代码逻辑（`accept()` 之后）时，**TCP 三次握手已经完成了**。半连接队列在操作系统内核中就已经被填满了。代码还没运行，内核已经瘫痪了。

#### 2. “DDoS 攻击中，为什么攻击者喜欢伪造 IP？”
* **深度回答**：
 * **反射攻击 (Amplification)**：攻击者发一个 1 字节的 NTP/DNS 请求给第三方服务器，但伪造**源 IP 为受害者 IP**。第三方服务器会将 50 倍体积的响应发给受害者，实现“借刀杀人”且隐藏了自己。

#### 3. “如果用户使用了 CDN，DDoS 攻击会攻击到真实 IP 吗？”
* **避坑指南**：如果**真实 IP（源站）泄露**，攻击者可以绕过 CDN 直接轰炸源站。因此，隐藏源站 IP（只允许 CDN 段访问）是防御的关键。

### 💻 极客演示：如何在 Linux 上加固？

通过内核参数优化，可以极大提升单机抗压能力：

```bash
# 1. 开启 SYN Cookies (防御 SYN Flood)
sysctl -w net.ipv4.tcp_syncookies=1

# 2. 增加半连接队列长度
sysctl -w net.ipv4.tcp_max_syn_backlog=4096

# 3. 减少 SYN 重试次数，让坏连接尽快超时
sysctl -w net.ipv4.tcp_syn_retries=2
sysctl -w net.ipv4.tcp_synack_retries=2

# 4. 使用 iptables 进行简单限流
iptables -A INPUT -p tcp --dport 80 -m limit --limit 25/minute --limit-burst 100 -j ACCEPT
```

### 📝 一分钟总结 (Summary)

1. **DDoS** 是利用流量规模或协议漏洞压垮服务器。
2. **防御核心**：网络层靠 **SYN Cookie** 和 **高防清洗**；应用层靠 **限流** 和 **WAF**。
3. **误区**：DDoS 无法通过简单的代码优化根治，通常需要投入资金购买**带宽和专业服务**。
