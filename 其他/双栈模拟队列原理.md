使用两个栈（Stack）模拟队列（Queue）是一个经典的算法问题。其核心逻辑在于利用栈的**后进先出（LIFO）特性，通过两次翻转来实现队列的先进先出（FIFO）**。

### 核心原理

我们需要定义两个栈：

1.  **输入栈（**`stackIn`**）**：专门负责接收新插入的数据。
    
2.  **输出栈（**`stackOut`**）**：专门负责弹出或查看队首数据。
    

---

### 具体操作步骤

#### 1\. 入队（Push）

直接将元素压入 `stackIn` 即可。

-   **时间复杂度**：$O(1)$
    

#### 2\. 出队（Pop）与 查看队首（Peek）

这是实现 FIFO 的关键步骤：

-   如果 `stackOut` 为空，则将 `stackIn` 中的**所有**元素逐一弹出并压入 `stackOut`。
    
-   如果 `stackOut` 不为空，直接从 `stackOut` 弹出顶部元素。
    
-   如果两个栈都为空，则说明队列为空。
    
-   **时间复杂度**：均摊 $O(1)$。虽然单次移动元素可能是 $O(n)$，但每个元素最多只会被移动两次（入 `stackOut` 和出 `stackOut`）。
    

---

### 代码实现 (Java)

```java
import java.util.Stack;

public class MyQueue {
    private Stack<Integer> inStack;
    private Stack<Integer> outStack;

    public MyQueue() {
        inStack = new Stack<>();
        outStack = new Stack<>();
    }

    /** 入队操作 */
    public void push(int x) {
        inStack.push(x);
    }

    /** 出队操作 */
    public int pop() {
        checkOutStack();
        if (outStack.isEmpty()) {
            throw new RuntimeException("Queue is empty");
        }
        return outStack.pop();
    }

    /** 获取队首元素 */
    public int peek() {
        checkOutStack();
        if (outStack.isEmpty()) {
            throw new RuntimeException("Queue is empty");
        }
        return outStack.peek();
    }

    /** 判空 */
    public boolean empty() {
        return inStack.isEmpty() && outStack.isEmpty();
    }

    /** * 核心辅助方法：
     * 只有当 outStack 为空时，才将 inStack 的元素全部倒入 outStack
     */
    private void checkOutStack() {
        if (outStack.isEmpty()) {
            while (!inStack.isEmpty()) {
                outStack.push(inStack.pop());
            }
        }
    }
}
```

---

### 关键点总结

| 操作  | 逻辑  |
| --- | --- |
| **入队** | 始终进入 `stackIn`。 |
| **出队** | 优先从 `stackOut` 取；若其为空，则将 `stackIn` “倒吸”进去再取。 |
| **顺序转换** | 两次 LIFO（后进先出）等于一次 FIFO（先进先出）。 |

> **注意**：在 `pop` 或 `peek` 时，必须保证 `stackOut` 里的元素全部取完后，才能进行下一次从 `stackIn` 到 `stackOut` 的搬运，否则会打乱元素的先后顺序。
