
**策略模式（Strategy Pattern）** 是另一种非常常用的行为型设计模式。

如果说**责任链模式**是“多个对象接力处理请求”，那么**策略模式**就是“**根据不同情况，从一堆算法中挑一个来用**”。

---

### 1. 什么是策略模式？

**核心思想：** 定义一系列算法，把它们一个个封装起来，并且使它们可以相互替换。

* **目的：** 将算法的定义与使用分离，避免使用大量的 `if-else` 或 `switch-case`。
* **生活例子：** **收银减免**。
 * 策略 A：原价。
 * 策略 B：打 8 折。
 * 策略 C：满 300 减 50。
 * 结算时，你根据当前的活动**选择其中一种**计算方式即可。

#### 代码示例（Java）：
```java
// 1. 定义策略接口
interface PaymentStrategy {
    void pay(int amount);
}

// 2. 实现具体策略
class AliPay implements PaymentStrategy {
    public void pay(int amount) { System.out.println("使用支付宝支付：" + amount); }
}

class WeChatPay implements PaymentStrategy {
    public void pay(int amount) { System.out.println("使用微信支付：" + amount); }
}

// 3. 上下文环境（Context）
class ShoppingCart {
    private PaymentStrategy strategy;
    
    public void setStrategy(PaymentStrategy strategy) { this.strategy = strategy; }
    
    public void checkout(int amount) {
        strategy.pay(amount); // 调用具体的策略
    }
}
```

### 2. 策略模式 vs 责任链模式：异同点

这两个模式看起来都有“多个处理者”，但逻辑逻辑完全不同。

#### A. 核心区别（一句话总结）
* **策略模式**是：你已经知道要用哪种方式了，直接**选一个**。
* **责任链模式**是：你不知道最终谁会处理，你把请求**扔进链条**，让它们自己传。

#### B. 详细对比

| 维度 | 策略模式 (Strategy) | 责任链模式 (Chain of Responsibility) |
| :--- | :--- | :--- |
| **处理者数量** | **只有一个**策略会被执行。 | **一个或多个**处理者可能会执行。 |
| **选择权** | **客户端（用户）**通常决定使用哪个策略。 | **链条内部**决定请求如何传递，客户端只管发。 |
| **结构特点** | 各个策略之间是**平级**的，互相独立，没有联系。 | 各个处理者之间是**链式**的，持有下一个人的引用。 |
| **解决的问题** | 解决 `if-else` 过多，算法易变的问题。 | 解决请求发送者与多个潜在接收者之间的耦合问题。 |
| **逻辑流向** | **横向替换**：要么 A，要么 B。 | **纵向传递**：A 不行给 B，B 不行给 C。 |

### 3. 一个形象的比喻

#### 场景：你想去旅行
* **策略模式**：
 你面前有三个按钮：[飞机]、[火车]、[自行车]。
 你根据你的预算和时间，**主动选择**按下了 [飞机] 按钮。你直接坐飞机去了。
* **责任链模式**：
 你走到一个服务台，说“我要去旅行”。
 * 第一个柜台（自行车）：一看你的目的地有 2000 公里，说“我处理不了，给你转到火车柜台”。
 * 第二个柜台（火车）：一看票卖完了，说“我处理不了，给你转到飞机柜台”。
 * 第三个柜台（飞机）：说“没问题，我来帮你办”。

---

### 4. 它们能结合使用吗？

**能！** 这在复杂的业务系统中非常常见。

**案例：多级折扣系统**
1. 首先使用**责任链模式**：判断用户的身份（是普通会员？还是金卡？还是黑卡？）。请求在链条上传递，直到找到匹配的等级。
2. 找到等级后，使用**策略模式**：金卡会员可能有多种优惠可选（满减策略、积分抵扣策略），此时再根据用户选中的策略进行计算。

### 总结
* 如果你是在**一堆算法中选一个**执行，用**策略模式**。
* 如果你有一堆**步骤或层级审批**，且不确定具体谁处理，用**责任链模式**。
