代理模式（Proxy Pattern）和适配器模式（Adapter Pattern）在结构上非常相似——它们都属于**结构型设计模式**，并且都是通过一个\*\*包装类（Wrapper）\*\*去封装另一个对象。

但它们的**意图（Intent）和解决的问题**完全不同。

---

### 1\. 代理模式 (Proxy Pattern)

**核心思想：** 为其他对象提供一种代理，以**控制**对这个对象的访问。

-   **特点：** 代理类和被代理类实现的是**同一个接口**。
    
-   **目的：** 不改变接口，但要在访问实际对象前后**增加额外的逻辑**（如权限控制、日志记录、缓存、延迟加载等）。
    

#### 代码示例：

```java
interface Image { void display(); }

class RealImage implements Image {
    public void display() { System.out.println("显示图片内容"); }
}

class ProxyImage implements Image {
    private RealImage realImage;
    
    public void display() {
        if (realImage == null) {
            realImage = new RealImage(); // 延迟加载：只有真正需要时才创建
        }
        System.out.println("日志记录：准备显示图片");
        realImage.display();
    }
}
```

**生活案例：** 经纪人。你想找明星（RealSubject）唱歌，你必须先找经纪人（Proxy）。经纪人帮你谈合同、排档期，最后唱歌的还是明星。明星和经纪人都对外宣称“能唱歌”。

---

### 2\. 适配器模式 (Adapter Pattern)

**核心思想：** 将一个类的接口转换成客户希望的**另外一个接口**。

-   **特点：** 适配器类和被适配类通常实现的是**不同的接口**。
    
-   **目的：** 解决**不兼容**问题，让两个因为接口不匹配而无法在一起工作的类可以协同工作。
    

#### 代码示例：

```java
// 客户需要的接口（Type-C）
interface TypeC { void chargeWithTC(); }

// 现有的类（MicroUSB）
class MicroUSB {
    void chargeWithMicro() { System.out.println("使用MicroUSB充电"); }
}

// 适配器：把 MicroUSB 转换成 TypeC
class ChargeAdapter implements TypeC {
    private MicroUSB oldCharger = new MicroUSB();

    @Override
    public void chargeWithTC() {
        System.out.println("适配器转换中...");
        oldCharger.chargeWithMicro();
    }
}
```

**生活案例：** 电源适配器或转接头。你的手机充电线是 Type-C 的，但插座是三相插孔，你需要一个“适配器”把两者连起来。

---

### 3\. 代理模式 vs 适配器模式：深度对比

| 维度  | 代理模式 (Proxy) | 适配器模式 (Adapter) |
| --- | --- | --- |
| **核心意图** | **控制访问**。在不改变接口的前提下，增强功能或控制权限。 | **转换接口**。将不兼容的接口变得兼容。 |
| **接口关系** | 代理类与原类实现**相同**的接口。 | 适配器与原类实现**不同**的接口。 |
| **对象关系** | 代理类通常在内部自己创建/持有原对象。 | 适配器通常是把已有的对象传进来进行包装。 |
| **使用场景** | 1\. 远程代理（RPC）<br>2\. 权限校验<br>3\. 日志、事务（Spring AOP）<br>4\. 延迟加载 | 1\. 复用旧系统的功能<br>2\. 整合第三方库（接口不一致时）<br>3\. 跨平台兼容 |
| **你什么时候用** | 你想在调用某个方法时，“顺便”做点别的事。 | 你想调用某个类，但它的方法名或参数跟你要求的对不上。 |

---

### 4\. 总结：一句话区分

-   **代理模式**：你是我的代理人，你和我**长得一样**（接口相同），你帮我**挡掉**一些麻烦或者**记录**一下谁找过我。
    
-   **适配器模式**：你是个转换头，你和我**长得不一样**（接口不同），你帮我把我的“方头”**转换**成客户需要的“圆头”。
    

**补充：** 在 Java 领域，**代理模式**的应用极广，比如 Spring 的 **AOP**（面向切面编程）本质就是动态代理。而**适配器模式**多见于系统集成，比如 `InputStreamReader` 就是一个适配器，它把 `InputStream`（字节流）适配成了 `Reader`（字符流）。
