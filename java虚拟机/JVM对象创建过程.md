### 1\. 核心结论 (Core Conclusion)

在 Java 中，一个对象的创建（以 `new` 关键字为例）主要经历五个阶段： **类加载检查 -> 分配内存 -> 初始化零值 -> 设置对象头 -> 执行** `<init>` **方法**。

---

### 2\. 深度剖析 (Deep Breakdown)

#### 第一步：类加载检查 (Class Loading Check)

当 JVM 遇到一条 `new` 指令时，首先检查这个指令的参数是否能在常量池中定位到一个类的符号引用，并检查这个类是否已被加载、解析和初始化过。如果没有，必须先执行相应的**类加载过程**。

#### 第二步：分配内存 (Memory Allocation)

类加载检查通过后，JVM 将为新生对象分配内存（所需内存在类加载完成后便可完全确定）。

-   **分配方式：**
    
-   **指针碰撞 (Pointer Bumping)：** 适用于内存规整（没有碎片）的情况，如使用 Serial、ParNew 等带压缩整理过程的收集器。
    
-   **空闲列表 (Free List)：** 适用于内存不规整的情况，如使用 CMS 这种基于清除算法的收集器。
    
-   **并发安全问题：** 创建对象是非常频繁的操作，JVM 如何保证在多线程下分配内存不冲突？
    

1.  **CAS + 失败重试：** 保证更新操作的原子性。
    
2.  **TLAB (Thread Local Allocation Buffer)：** 为每个线程预先在 Eden 区分配一块私有内存。只有 TLAB 用完需要重新申请时，才需要同步锁定。
    

#### 第三步：初始化零值 (Zero Initialization)

内存分配完成后，JVM 将分配到的内存空间（不包括对象头）都初始化为零值（如 `int` 为 0，`boolean` 为 `false`，引用类型为 `null`）。

-   **作用：** 保证了对象的实例字段在 Java 代码中可以不赋初值就直接使用。
    

#### 第四步：设置对象头 (Setting Object Header)

JVM 对对象进行必要的设置，例如：

-   这个对象是哪个类的实例（元数据指针）。
    
-   对象的哈希码（HashCode）。
    
-   对象的 GC 分代年龄。
    
-   锁状态标志。
    
-   如果是数组，还需记录数组长度。
    

#### 第五步：执行 `<init>` 方法 (Initialization)

此时，从 JVM 角度看，对象已经产生了。但在 Java 程序角度，初始化才刚刚开始。

-   执行构造代码块、成员变量显式赋值。
    
-   执行**构造方法**。 按照程序员的意愿对对象进行真正的初始化。

### 3\. 面试官视角 (Interviewer's Lens)

**经典追问一：对象一定分配在堆上吗？**

-   *答案：* **不一定。** 经过 **逃逸分析 (Escape Analysis)**，如果发现一个对象不会逃逸出方法，JVM 可能会进行**栈上分配**或者**标量替换**，从而减轻堆内存和 GC 的压力。
    

**经典追问二：new 指令在字节码层面是什么样的？**

-   *答案：* 对应三条指令：
    

1.  `new`：分配内存、创建引用、初始化零值。
    
2.  `dup`：复制栈顶引用（一份用于 `<init>`，一份用于赋值）。
    
3.  `invokespecial`：调用构造方法。 *注意：* 这也是为什么单例模式 DCL 要加 `volatile` 的原因——防止指令重排序导致拿到了一个“半初始化”的对象。
    

**典型陷阱 (Pitfalls)：**

-   千万不要说“先执行构造方法再分配内存”，顺序是完全反的。
    
-   分不清“零值初始化”和“显式赋值”的区别。

### 5\. 总结 (Summary)

1.  **检查：** 先看类有没有加载。
    
2.  **分配：** 在堆里挖一块肉（注意 TLAB 性能优化）。
    
3.  **清零：** 所有人底色刷白（默认值）。
    
4.  **戴帽：** 扣上对象头（身份信息）。
    
5.  **初始化：** 填入程序员想要的颜色（构造函数）。
